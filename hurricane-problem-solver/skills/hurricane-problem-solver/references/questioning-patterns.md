# Questioning Patterns

Use this file when the problem is still fuzzy, when the visible complaint feels too neat, or when the investigation needs more pressure from unusual angles.

## 1. Complaint vs actual problem

Use these when the user is describing pain, but not yet the real problem.

- What exactly is happening?
- What is the complaint on the surface?
- What is the actual pain underneath it?
- What is inconvenient, and what is truly damaging?
- What changed that made this impossible to ignore now?
- If I remove the emotion and tell the story in plain facts, what remains?
- What would a clean one-line problem statement be right now?
- What would make that statement wrong or incomplete?

## 2. Time and chain-of-events pressure

Use these to expose drift, accumulation, and butterfly effects.

- When did this first become true?
- When did it first become painful?
- Are those the same date? Usually not.
- What happened right before that?
- What was already weakening before the visible break?
- Is this one incident, or the final visible crack in a long chain?
- What sequence had to happen for this symptom to emerge?
- Which step in that sequence quietly changed everything?

## 3. Pattern and non-pattern

Use these to separate cause from noise.

- Where does the problem happen consistently?
- Where does it not happen?
- What is different about those cases?
- What variable changes when the outcome changes?
- What stays weirdly constant across all cases?
- Which pattern are people normalizing because they see it every day?

## 4. Incentives, power, and resistance

Use these when human behavior is shaping the mess.

- Who benefits from this pattern staying alive?
- Who pays for it?
- Who has the power to change it but does not?
- What behavior is the system rewarding?
- What behavior is the system punishing?
- What does everyone complain about but continue to participate in?
- What would someone have to lose if this were fixed properly?

## 5. Constraint hunting

Use these when the team keeps acting strangely for reasons nobody says out loud.

- What constraint is everyone silently working around?
- What are people treating as fixed that may not actually be fixed?
- What resource is too thin: time, money, skill, attention, trust, ownership?
- What hidden dependency keeps forcing bad choices?
- What simple missing capability is making everything harder than it needs to be?

## 6. Symptom pretending to be cause

Use these when explanations sound polished but weak.

- Is this a cause, or just a better-worded symptom?
- What has to exist upstream for this to be true?
- If I fix this, does the rest collapse, or does the mess simply reappear elsewhere?
- What is feeding this issue energy?
- What allows it to persist?
- What would make this issue disappear without touching it directly?

## 7. Simplicity checks

Use these when the problem looks huge and theatrical.

- What is the smallest broken thing that could create all this drama?
- Are we staring at five issues that are actually one issue in a trench coat?
- What 20 percent of causes is producing 80 percent of the pain?
- If I had to bet on one leverage point, what would it be?
- What fix would create a domino effect if it were done properly?

## 8. Emotional / behavioral loops

Use these when the mess includes people, identity, fear, ego, or habits.

- What feeling is driving the behavior?
- What fear keeps the pattern alive?
- What identity would be threatened if this changed?
- What are people avoiding naming directly?
- What is everyone reacting to, instead of addressing?
- What loop keeps recreating the same result?

## 9. Elimination discipline

Use these to narrow the field without becoming dogmatic.

- Which live explanations still explain most of the symptoms?
- Which ones only explain one corner of the mess?
- Which explanation best predicts timing and recurrence?
- Which explanation survives contradictory evidence?
- Which explanations can now be killed cleanly?

## 10. Stopping rule

Stop digging when:

- one cause, or one tightly linked cluster, explains most of the field
- deeper explanation no longer changes what the leverage point is
- the remaining detail is refinement, not a shift in truth
- the intervention point becomes obvious enough to act on
